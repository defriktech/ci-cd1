# CI/CD Pipeline for Node.js Application 🚀

This project demonstrates a fully automated CI/CD pipeline for a Node.js application, showcasing modern DevOps best practices using **GitHub Actions**, **Docker Hub**, and **Google Cloud Run**.

## 🔧 What’s Included

- **Continuous Integration**: Automated workflows using GitHub Actions to test and integrate code changes upon each push.
- **Containerization**: The application is packaged into a Docker image and pushed to Docker Hub.
- **Staging Deployment**: Every code change is deployed to a staging environment via Google Cloud Run for testing and validation.
- **Continuous Deployment**: Production deployments are triggered only when a release is created from the `main` branch, ensuring safe and controlled updates.

## ✅ Key Benefits

- Enables early testing of new features without impacting the production environment.
- Promotes a robust DevOps workflow with minimal manual intervention.
- Makes deployment smoother, faster, and more reliable.

## 📦 Tech Stack

- Node.js
- GitHub Actions
- Docker & Docker Hub
- Google Cloud Run
